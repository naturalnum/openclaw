📋 项目概览

文件加密插件为 OpenClaw 提供透明的文件加密/解密功能，保护敏感的配置文件和记忆文件。使用业界标准的 AES-256-GCM 加密算法，确保数据的机密性和完整性。

核心优势：
  • 🔐 AES-256-GCM 业界标准加密算法
  • 🔄 自动加密/解密，对应用透明
  • 📁 支持 glob 模式选择文件
  • 🎛️ 多种操作模式：encrypt / decrypt / both
  • 🔑 环境变量优先，安全可靠

✨ 核心功能

默认加密路径

路径模式说明
**/memory/**所有记忆文件
**/.openclaw/config.*配置文件
**/sessions/**会话数据
**/*.secret.*秘密文件

操作模式

  • encrypt - 仅加密（写入时）
  • decrypt - 仅解密（读取时）
  • both - 双向（默认）

🚀 快速开始

步骤 1: 生成加密密钥

openssl rand -base64 32

步骤 2: 设置环境变量

export OPENCLAW_FILE_ENCRYPTION_KEY="你的密钥"

步骤 3: 配置插件

{
  "plugins": {
    "entries": {
      "file-encryption-plugin": {
        "enabled": true
      }
    }
  }
}

步骤 4: 重启 OpenClaw

pkill -9 -f openclaw-gateway || true
openclaw gateway run --bind loopback --port 18789

⚙️ 配置指南

基础配置

{
  "plugins": {
    "entries": {
      "file-encryption-plugin": {
        "enabled": true,
        "config": {
          "mode": "both",
          "encryptedPaths": [
            "**/memory/**",
            "**/.openclaw/config.*",
            "**/sessions/**"
          ]
        }
      }
    }
  }
}

配置选项

选项类型默认值说明
encryptionKeystring环境变量AES-256 密钥 (base64)
modestring"both"操作模式
encryptedPathsstring[]内置默认要加密的路径模式

🔐 安全管理

⚠️ 重要安全提示：
  • 永远不要将加密密钥提交到版本控制
  • 使用环境变量存储密钥（推荐）
  • 定期轮换加密密钥
  • 备份密钥到安全位置（密码管理器）
  • 丢失密钥 = 永久数据丢失

密钥生成方式

方式 1: 随机密钥（推荐）

openssl rand -base64 32

方式 2: 密码派生（PBKDF2）

node derive-key-from-password.js "your-memorable-password"

🔓 自动解密功能

OpenClaw 的 read 工具已集成自动解密功能，读取加密文件时会自动将密文解密为明文返回给用户。

✨ 核心特性：
  • 🔄 透明操作 - 用户无感知，自动识别和解密
  • 🔐 安全可靠 - AES-256-GCM 解密算法
  • ⚡ 优雅降级 - 解密失败不影响正常文件读取
  • 🎯 精准识别 - 只解密符合格式的内容

工作原理

用户请求读取文件
    ↓
read 工具执行
    ↓
读取文件内容（可能是密文）
    ↓
检查环境变量 OPENCLAW_FILE_ENCRYPTION_KEY
    ↓
如果密钥存在：
    ├─ 检查内容是否为加密格式 (iv:authTag:content)
    ├─ 如果是加密内容：
    │   ├─ 使用 AES-256-GCM 解密
    │   ├─ 返回解密后的明文 ✅
    │   └─ 记录调试日志（如果失败）
    └─ 如果不是加密内容：
        └─ 返回原始内容
    ↓
返回结果给用户

加密内容格式

系统只识别符合以下格式的内容：

base64(iv):base64(authTag):base64(encryptedContent)

例如：

YWJjZGVmZ2hpamtsbW5vcA==:cXJzdHV2d3h5ejAxMjM0NTY=:ZW5jcnlwdGVkIGNvbnRlbnQ=

完整工作流程

步骤组件操作
1file-encryption-plugin拦截 write 操作
2file-encryption-plugin加密文件内容
3文件系统写入密文到磁盘
4read 工具读取文件内容
5pi-tools.read.ts检测是否为密文
6pi-tools.read.ts自动解密为明文
7用户看到明文内容 ✅

安全特性

1. 环境变量优先

  • 密钥从 OPENCLAW_FILE_ENCRYPTION_KEY 环境变量读取
  • 不会硬编码在代码中
  • 支持不同环境使用不同密钥

2. 优雅降级

  • 如果密钥未设置，直接返回原始内容
  • 如果解密失败，捕获异常并返回原始内容
  • 不会影响正常文件的读取

3. 透明操作

  • 对上层应用完全透明
  • 用户无需手动解密
  • 自动识别加密/明文内容

代码实现位置

自动解密功能实现在 OpenClaw 核心代码中：

文件：src/agents/pi-tools.read.ts
函数：createOpenClawReadTool()
位置：execute 方法中的自动解密逻辑

配置要求

⚠️ 重要提示：

要启用自动解密功能，必须设置环境变量：

export OPENCLAW_FILE_ENCRYPTION_KEY="你的32字节密钥"

如果未设置该环境变量，read 工具将返回原始文件内容（密文）。

示例场景

场景 1: 读取加密的记忆文件

~/.openclaw/workspace/memory/notes.md

# 用户通过 read 工具读取

# 用户通过 read 工具读取

💡 最佳实践：
  • 确保环境变量正确设置

  • 定期测试解密功能是否正常
  • 查看日志确认解密状态：openclaw logs | grep auto-decryption

🆘 故障排查

问题 1: "No encryption key provided"

解决方案：

export OPENCLAW_FILE_ENCRYPTION_KEY=$(openssl rand -base64 32)

问题 2: "Invalid key length"

解决方案：密钥必须是 32 字节，使用 OpenSSL 生成

openssl rand -base64 32

问题 3: 文件没有被加密

检查清单：

  • 插件已启用 ("enabled": true)
  • 文件路径匹配 encryptedPaths 模式
  • 加密密钥有效
  • 查看日志：openclaw logs | grep file-encryption-plugin
