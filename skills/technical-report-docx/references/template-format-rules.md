# Template Format Rules

Use these rules when transferring formatting from the fixed technical-report template to a new DOCX.

## Preserve

- style names and style roles when they are available
- heading/body/list/table distinctions
- paragraph spacing before/after
- line spacing and indentation
- font family choices for headings and body text
- emphasis patterns such as bold inline labels
- section breaks and page setup
- cover layout rhythm
- figure/table caption style

## Prefer

- applying template styles over direct formatting
- replacing text inside existing styled paragraphs when safe
- using the template as the base document when it contains reusable document structure

## Avoid

- copying source-document inline formatting into the final output
- mixing source font choices with template font choices
- leaving original template sample text visible
- flattening all text into one generic style
- treating a technical report like a generic memo or proposal

## Validation Checklist

- heading levels look consistent
- body paragraphs use the template body style
- tables do not introduce foreign fonts
- headers and footers still match the template
- no page contains clipped or overlapping text
- the report still feels like the canonical technical-report template
